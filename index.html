Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I would tell them the following:

  • Implement a strong password policy.
  • Enable two-factor authentication if posbile.
  • Continuous Integration/CI is a greate way to do secuirty checks before it goes out. If it get flaged then you can rework it.
  • Install a web application firewall. This will protect against common web-based attacks such as SQL injections and cross-site scripting
  • Implementing secure protocols. This includes using secure protocols such as SSL/TLS and SSH when transferring data between systems.
  • Only use well-known thrid party applications.
  • Use NodeJS sanitize-html package. To sanitize HTML code.
  • Use "https" so everything is encrypted, and attackers can't get your info.
  • Make sure your API is working corretly.
  • Don't check your private email on work.
  • Keep important files local if posbile
  • Get a good IT team.
  • Monitoring and logging. All website activity should be monitored and logged, and any suspicious activity should be investigated
  • Remember to take backups of your website code base
  • Remember to update your software.
  • You and your crew should always have secuirty in mind by default.
